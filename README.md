## 🎯 项目介绍

**VoiceDraw AI** 是一款专为语音交互场景设计的数据可视化与绘图工具。用户全程无需使用鼠标或键盘，仅通过**中文语音指令**即可完成从图表创建（柱状图/饼图/折线图/流程图）到基础图形绘制、颜色填充、样式调整、导出等完整创作流程。

项目已从最初的纯绘图工具升级为 **v6.0.0 Voice DataViz Tool**，核心聚焦于语音驱动的数据可视化图表创建与编辑，同时保留基础绘图能力。

### 项目亮点

| 维度 | 实现方式 |
|------|---------|
| **零依赖** | 纯浏览器原生API，无任何第三方框架/CDN |
| **即开即用** | 单HTML文件，双击即可运行 |
| **中文优化** | 针对中文语音特点进行关键词提取与模糊匹配 |
| **图表向导** | 分步引导式语音创建图表（数量→数值→标签→标题） |
| **多风格主题** | 暖色风/商务风/科技风/简约风四种图表风格 |
| **容错设计** | 支持多种近义表达、模糊匹配、多候选结果 |
| **即时反馈** | 语音合成(TTS)即时确认操作结果 |

## ✨ 核心功能

### 🎤 语音识别模块
- 基于 `Web Speech API (SpeechRecognition)` 的实时中文语音识别
- 持续聆听模式，自动重启识别引擎
- 中间结果实时预览（画布底部覆盖层展示）
- 多候选结果择优匹配（最多3个候选）
- 语音识别错误自动恢复与友好提示

### 📊 数据可视化图表
- **柱状图**: 语音分步创建 → 编辑柱子颜色/宽度/标签/标题
- **饼图**: 语音分步创建 → 编辑扇区颜色/标签/标题
- **折线图**: 语音分步创建 → 编辑数据点/标签/标题/线型
- **流程图**: 语音分步创建 → 编辑节点名称/节点框样式/箭头样式

### 🎨 图表风格主题
- 🔥 **暖色风**: 橙色系配色，白色背景
- 💼 **商务风**: 蓝色系配色，白色背景
- 🚀 **科技风**: 霓虹配色，深色背景
- 📐 **简约风**: 灰黑色系配色，白色背景
- 四种风格均适配柱状图、饼图、折线图、流程图
- 流程图四种风格自适应节点配色和统一边框颜色

### 🧠 图表创建向导
- 分步引导：数量 → 数值 → 标签 → 标题（可选）
- 支持柱状图、饼图、折线图、流程图四种类型
- 语音输入每个数据值/标签/标题
- 完成后自动提供修改指令提示

### ✏️ 图表编辑功能
- 修改指定柱子/扇区颜色
- 调整标题/标签字号（放大/缩小/加粗）
- 修改标签文字内容
- 批量修改标签颜色
- 柱子粗细调整
- 流程图节点名称/节点框/箭头样式编辑
- 图表类型专属修改指令列表自动切换

### 🖌️ 基础绘图画布
- 自由绘制模式（画笔）
- 直线绘制 / 圆形/椭圆绘制 / 矩形/正方形绘制
- 洪水填充（区域填色，4连通域算法）
- 橡皮擦模式
- 10种预设颜色 + 线宽动态调整（1-20px）

### ⏱️ 操作控制
- 撤销（操作历史栈，支持图表撤销跨步骤恢复）
- 清空画布 / 重置所有设置
- 导出PNG图片（3x高清导出，保持当前风格不变）
- 键盘快捷键（Ctrl+Z 撤销, Ctrl+S 导出, Esc 清空）

### 🔊 语音反馈
- 基于 `SpeechSynthesis API` 的TTS中文语音反馈
- 操作成功/失败差异化提示
- 高/普通优先级语音队列管理

### 📋 辅助功能
- 指令日志实时显示（最多50条）
- 状态栏统计（识别状态、图形数、指令数）
- 可用指令提示条（自动显示/折叠）
- 修改指令提示条（图表创建完成后按类型自动切换）
- 触控屏支持
- 响应式布局（适配768px以下屏幕）

## 🚀 快速开始

### 运行方式

**方式一：直接打开（推荐）**
1. 下载 `index.html` 文件
2. 使用 **Google Chrome** 浏览器双击打开
3. 点击麦克风按钮 🎙️
4. 允许浏览器使用麦克风权限
5. 开始说出指令！如"画柱状图"、"画饼图"、"画折线图"

**方式二：本地服务器**
```bash
# Python 3
python -m http.server 8080

# Node.js (npx)
npx serve .

# 然后访问 http://localhost:8080
```

### 环境要求

| 要求 | 说明 |
|------|------|
| **浏览器** | Google Chrome 80+（推荐）、Microsoft Edge 80+ |
| **权限** | 麦克风访问权限 |
| **网络** | 首次使用时需要网络连接（Chrome语音识别依赖云端服务） |

> ⚠️ **重要**: Web Speech API 的语音识别功能在 Chrome 浏览器中需要网络连接。Firefox 和 Safari 目前对中文语音识别的支持有限。

## 📋 使用说明

### 图表创建流程（推荐）

1. **启动识别**: 点击麦克风按钮
2. **说出图表类型**: "画柱状图" / "画饼图" / "画折线图" / "画流程图"
3. **回答数量**: 系统会询问"要画几根柱子？"，说出数字如"5"
4. **逐个输入数值**: 系统依次询问每个柱子的数值
5. **选择标签**: 需要时添加标签名称
6. **设置标题**: 需要时设置图表标题
7. **语音编辑**: 使用修改指令调整图表样式

### 基础绘图流程

1. **启动识别**: 点击麦克风按钮
2. **选择模式**: 说"画圆" / "画矩形" / "自由画"等
3. **选择颜色**: 说"红色" / "蓝色"等
4. **开始绘制**: 在画布上点击并拖动鼠标
5. **调整样式**: 说"加粗" / "变细"
6. **撤销修改**: 说"撤销"
7. **导出作品**: 说"导出"

### 核心技术选型

| 模块 | 技术 | 选择理由 |
|------|------|---------|
| 语音识别 | `SpeechRecognition` (Web Speech API) | 浏览器原生支持，中文识别准确率较高 |
| 语音合成 | `SpeechSynthesis` (Web Speech API) | 与识别同源API，提供中文TTS反馈 |
| 绘图渲染 | `Canvas 2D API` | 浏览器原生，性能优异，支持像素级操作 |
| 撤销机制 | 操作历史栈 + ImageData快照栈 | 双栈机制，支持图表和自由绘制的完整撤销 |
| 图表渲染 | 纯Canvas 2D自绘（柱/饼/线/流程） | 零依赖图表的高性能实现 |
| 填充算法 | 洪水填充 (Flood Fill) | 经典4连通域算法，基于像素颜色容差匹配 |
| 指令解析 | 关键词匹配 + 字符重叠度模糊匹配 | 轻量级NLP方案，响应即时 |
| 图表向导 | 状态机模式 (Step-based FSM) | 管理分步创建流程（数量→数值→标签→标题） |
| 图表编辑 | chartConfig 配置对象 + 增量重绘 | 支持逐元素样式修改（颜色/大小/粗细） |

## 📦 依赖列表

### 运行时依赖

**本项目为零外部依赖（Zero-Dependency）纯浏览器原生应用。**

所有功能均基于以下浏览器原生API实现：

| API | 用途 | W3C标准 |
|-----|------|---------|
| `SpeechRecognition` | 语音识别（中文zh-CN） | [Web Speech API](https://wicg.github.io/speech-api/) |
| `SpeechSynthesis` | 语音合成反馈（中文zh-CN） | [Web Speech API](https://wicg.github.io/speech-api/) |
| `Canvas 2D Context` | 图表渲染 + 绘图渲染 | [HTML Canvas 2D Context](https://html.spec.whatwg.org/multipage/canvas.html) |
| `ImageData` | 像素级画布操作（撤销/填充/导出） | [HTML Living Standard](https://html.spec.whatwg.org/) |
| `fetch` | HTTP请求（AI预留接口） | [Fetch API](https://fetch.spec.whatwg.org/) |

### 开发/构建依赖

**无。** 本项目为纯静态HTML文件，无需编译、打包、转译。
